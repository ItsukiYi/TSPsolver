OVERALL PROJECT PIPELINE ARCHITECTURE — FOR FIGURE GENERATION
==============================================================

This document describes the complete architecture of our TSP solver project.
It covers: (A) all algorithms studied, (B) the DIFUSCO→DualOpt hybrid pipeline
(our key contribution), (C) the evaluation framework, and (D) visual guidance.

======================================================================
A. ALGORITHM LANDSCAPE (6 methods, from classic to AI to gold standard)
======================================================================

We study 6 methods spanning 4 paradigms:

[1] Nearest Neighbor (CLASSIC — CONSTRUCTION)
  Input: n node coordinates in [0,1]^2
  Process: Start at depot -> repeatedly visit nearest unvisited node -> return
  Output: A valid TSP tour (usually 20-30% worse than optimal)
  Complexity: O(n^2)
  Role: Simplest baseline, defines the "floor"

[2] Christofides Algorithm (CLASSIC — APPROXIMATION)
  Input: n node coordinates
  Process:
    a) Build Minimum Spanning Tree (MST)
    b) Find odd-degree vertices in MST
    c) Compute Minimum-Weight Perfect Matching on those vertices (Blossom alg)
    d) Combine MST + matching edges -> Eulerian multigraph
    e) Extract Eulerian circuit -> shortcut to Hamiltonian cycle
  Output: Valid TSP tour with theoretical guarantee: cost <= 1.5 × OPT
  Complexity: O(n^3)
  Role: Strong theoretical baseline

[3] C+2opt (CLASSIC — IMPROVEMENT)
  Input: n node coordinates
  Process: Christofides (Step 2 above) -> 2-opt local search polish
  Output: Near-optimal tour (gap ~3-5% on TSP-50)
  Complexity: O(n^3)
  Role: Strongest classical baseline in our study

[4] DIFUSCO+2opt (AI — GENERATIVE)
  Input: n node coordinates
  Process:
    a) Random Bernoulli noise matrix (n×n)
    b) 50-step categorical denoising via 12-layer GNN (5.3M params)
    c) Output: edge probability heatmap H[i][j] in [0,1]
    d) Greedy merge: rank edges by H[i][j]/distance(i,j), build tour
    e) GPU-accelerated 2-opt polish (1000 iterations)
  Output: Valid TSP tour
  Complexity: O(n^2 · d^2), d=256
  Training: Supervised, cross-entropy loss on edge labels
  Role: Primary reproduction target (Sun & Yang, NeurIPS 2023)

[5] DualOpt (AI — IMPROVEMENT)
  Input: An initial TSP tour (from C+2opt or other solver)
  Process: Sliding-window neural reviser with 3 scales:
    a) k=50 window: coarse revision, 25 iterations
    b) k=20 window: medium revision, 10 iterations
    c) k=10 window: fine polish, 5 iterations
    Each reviser is an attention-based policy network (~710K params)
    trained via REINFORCE to minimize tour length
  Output: Refined tour (gap ~2-3% better than input)
  Key property: INFERENCE TIME IS CONSTANT (~6 seconds) regardless of
    instance size (n=50 to n=500), because each window is always 50 nodes
  Training: Reinforcement learning (REINFORCE with rollout baseline)
  Role: Secondary reproduction target (Zhou et al., AAAI 2025)

[6] LKH3 (GOLD STANDARD)
  Input: Any initial tour
  Process: k-opt search (k=2,3,4,5) guided by α-nearness from minimum 1-tree
  Output: Near-optimal tour (gap <0.5%, often exact optimum)
  Complexity: ~O(n^2.2) empirically
  Implementation: Pure C, highly optimized
  Role: Ceiling — defines what's achievable. Always the fastest AND best
    quality method at every scale we tested (TSP-50 to TSP-1000).

======================================================================
B. DIFUSCO -> DUALOPT HYBRID PIPELINE (OUR KEY CONTRIBUTION)
======================================================================
This is Improvement #2 — the ONLY improvement that successfully beats
the baseline. It combines the generative power of diffusion with the
local optimization power of neural revisers.

PIPELINE ARCHITECTURE (Two Phases):

PHASE 1: DIFFUSION CONSTRUCTION (DIFUSCO)
  Step 1: Input = n node coordinates (x_i, y_i)
  Step 2: Initialize random Bernoulli noise matrix X_T (size n×n, each
          entry independently 0 or 1 with 50% probability)
  Step 3: 50-step denoising loop (t = T, T-1, ..., 1):
          - GNN takes (coordinates, noisy adjacency X_t, timestep t)
          - Predicts clean adjacency X_0_hat
          - Computes X_{t-1} via categorical posterior
          - Cosine noise schedule, DDIM acceleration
  Step 4: After 50 steps, obtain edge heatmap H[i][j] in [0,1]
          (probability that edge (i,j) belongs to a good tour)
  Step 5: Greedy merge algorithm:
          - Score each possible edge: H[i][j] / Euclidean_distance(i,j)
          - Sort edges by score (descending)
          - Insert edges into an empty graph if they don't create
            degree-3 nodes or sub-cycles (except the final Hamiltonian cycle)
          - Result: a valid TSP tour (initial tour π_0)
  Step 6 (optional): 2-opt polish on π_0 (1000 iterations, GPU-accelerated)

PHASE 2: NEURAL REFINEMENT (DualOpt Reviser)
  Step 7: Take initial tour π_0 (from Phase 1 or from the optional 2-opt)
  Step 8: Apply reviser k=50:
          - Decompose tour into non-overlapping windows of 50 nodes
          - For each window: neural reviser proposes improved sub-tour ordering
          - Reassemble windows into full tour
          - Repeat 25 times with different window starting positions
  Step 9: Apply reviser k=20 (10 iterations, same sliding-window logic)
  Step 10: Apply reviser k=10 (5 iterations, finest polish)
  Step 11: Output = final optimized tour π*

PERFORMANCE (TSP-50, 10 test instances):
  DIFUSCO raw (no polish):          6.696  (+13.67% vs GT)
  DIFUSCO + 2-opt (1000 iterations): 5.969  (+1.32% vs GT)
  DIFUSCO -> DualOpt (our pipeline): 5.793  (-1.67% vs GT) <-- BEST
  Improvement over DIFUSCO raw: -13.50% (the DualOpt reviser extracts
  24% more improvement than 2-opt alone from the same heatmap)

CRITICAL FINDING — PER-SCALE TRAINING REQUIREMENT:
  TSP-50  + DIFUSCO trained on TSP-50  -> Pipeline works (-1.67%)
  TSP-100 + DIFUSCO trained on TSP-50  -> Pipeline FAILS (-2.24%)
  TSP-100 + DIFUSCO trained on TSP-100 -> Pipeline works (+1.56%)
  Conclusion: DIFUSCO MUST be trained at the target scale.
  Cross-scale transfer does not work because the heatmap quality degrades.

======================================================================
C. EVALUATION FRAMEWORK
======================================================================

Datasets Tested (3 categories):
  1. Random Uniform TSP: n = 50, 100, 200, 500, 1000
     (same as DIFUSCO paper's training distribution)
  2. TSPLIB Standard Benchmark: 8 instances (51 to 1002 nodes)
     with known optimal values from the literature
     (Neither paper fully covered all these instances)
  3. Clustered Delivery TSP: 4 Gaussian clusters + scattered points
     at n = 50, 100, 200
     (NEITHER original paper tested on clustered data — our novel contribution)

Metrics:
  - Tour cost (lower is better)
  - Gap vs Ground Truth or vs LKH3 (%)
  - Runtime per instance (seconds)
  - Pareto frontier: speed vs quality tradeoff

Baseline Comparisons:
  Every method compared against: NN, Christofides, C+2opt, DIFUSCO+2opt,
  DualOpt, LKH3. This gives a complete picture from simplest to gold standard.

======================================================================
D. KEY FINDINGS (for annotation on diagrams)
======================================================================

1. DIFUSCO → DualOpt pipeline is the ONLY effective improvement
   (5 improvements attempted, 4 failed, 1 succeeded)

2. The DualOpt reviser alone (without grid divide-and-conquer) works on any
   scale (TSP-50 to TSP-500) in constant ~6 second inference time

3. DIFUSCO heatmap confidence is NOT calibrated:
   - On TSP-50 (in-distribution): only 45% of high-confidence edges are optimal
   - On TSPLIB (out-of-distribution): 0% are optimal
   This explains why all heatmap-guided improvements (#1, #4, #5) failed

4. LKH3 remains unbeaten at every scale we tested (TSP-50 to TSP-1000).
   The DualOpt paper's claim of surpassing LKH3 applies at TSP-100K scale
   which we cannot test due to hardware limits.

5. TSP-50 is "solved" for DualOpt — its reviser converges to a local
   optimum that no post-processing (heatmap guidance, 2-opt diagnostic,
   edge freezing, or destroy-and-repair) can escape
